markdown 规范 #

01 标题原则 #

  1. 一级标题下，不能直接出现三级标题
  2. 标题要避免孤立编号
  3. 下级标题不重复上一级标题的名字
  4. 谨慎使用四级标题，尽量避免出现，保持层级的简单，防止出现过于复杂的章节

02 字间距 #

  1. 全角中文字符与半角英文字符之间，应有一个半角空格
  2. 全角中文字符与半角阿拉伯数字之间，有没有半角空格都可，但是必须保证风格统一
  3. 英文单位若不翻译，单位前的阿拉伯数字与单位符号之间，应留出适当的空隙
  4. 半角英文字符和半角阿拉伯数字，与全角标点符号之间不留空格

03 句子 #

  1. 避免使用长句。不包含任何标点符号的单个句子，或者以逗号分隔的句子构件，长度尽量保持在20个字符。
  2. 尽量使用简单句和并列句，避免使用复合句。
  3. 同样一个意思，尽量使用肯定句表达，不使用否定句表达。
  4. 避免使用双重否定句。

04 写作风格 #

  1. 尽量不使用被动语态，改为使用主动语态。
  2. 不使用非正式的语言风格。
  3. 不使用冷僻、生造或文言文的词语，而要使用现代汉语的常用表达方式。
  4. 用对“的”、“地”、“得”
  5. 使用代词时（比如“其”、“该”、“此”、“这”等词），必须明确指代的内容，保证只有一个含义。
  6. 名词前不要使用过多的形容词。

05 英文处理 #

  1. 英文原文如果使用了复数形式，翻译成中文时，应该将其还原为单数形式。
  2. 外文缩写可以使用半角圆点（.）表示缩写
  3. 表示中文时，英文省略号（...）应改为中文省略号（……）。
  4. 英文书名或电影名改用中文表达时，双引号应改为书名号。
  5. 第一次出现英文词汇时，在括号中给出中文标注，此后再次出现时，直接使用英文缩写即可。
  6. 专有名词中每个词第一个字母均应大写，非专有名词则不需要大写。

06 段落 #

  1. 一个段落只能有一个主题，或一个中心句子。
  2. 段落的中心句子放在段首，对全段内容进行概述。后面陈述的句子为核心句服务。
  3. 一个段落的长度不能超过七行，最佳段落长度小于等于四行。
  4. 段落的句子语气要使用陈述和肯定语气，避免使用感叹语气。
  5. 段落之间使用一个空行隔开。
  6. 段落开头不要留出空白字符。

07 引用 #

  1. 引用第三方内容时，应注明出处。
  2. 如果是全篇装载，请在全文开头显著位置注明作者和出处，并链接至原文。
  3. 使用外部图片时，必须在图片下方或文末标明来源。

08 数值 #

  1. 半角数字。阿拉伯数字一律使用半角形式，不得使用全角形式。
  2. 千分号。数值为千位以上，应添加千分号（半角逗号）。
  3. 货币。货币应为阿拉伯数字，并在数字前写出货币符号，或在数字后写出货币中文名称。
  4. 数值范围。表示数值范围时，用波浪线（~）或一字线（-）连接。
  5. 带有单位或百分号时，两个数字建议都要加上单位或百分号。
  6. 变化程度的表示法。
    • 数字的增加要使用“增加了”、“增加到”。“了”表示增量，“到”表示定量。
    • 数字的减少要使用“降低了”、“降低到”、“了”表示增量，“到”表示定量。
    • 不能用“降低N倍”或“减少N倍”，要用“降低百分之几”或“减少百分之几”。

09 标点符号 #

  1. 中文语句的结尾处应该用全角句号（。）。
  2. 句子末尾用括号加注时，句号应在括号之外。
  3. 引号里面还要引号时，外面一层用双引号，里面一层用单引号（‘ ’），注意前后单引号不同。 例句：鲍勃解释道：“我要放音乐，可萨利说，‘不行！’。”
  4. 破折号一般用于进一步解释。

10 文件 #

  1. 文档的文件名不得含有空格。
  2. Github 不能用中文命名。
  3. 文件名建议只使用小写字母，不使用大写字母。
  4. 文件名包含多个单词时，单词之间建议使用半角的连词线（-）分隔。
