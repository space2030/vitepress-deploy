java 开发规范 #

01 不正确的代码实现 #

  1. 不对等。业务逻辑关系不对等。
  2. 部分抽取。对整体的部分逻辑进行抽取，这种方法很难命名，影响代码阅读和理解。
  3. 抽取错误。和逻辑不匹配。
  4. 补丁代码。
  5. 先梳理清楚逻辑树，然后做到代码符合的逻辑树。

03 迭代entrySet()获取Map的key和value #

反例

// Map 获取 value 反例:
HashMap<String, String> map = new HashMap<>();
for (String key : map.keySet()){
String value = map.get(key);
}

正例

//Map 获取 key & value 正例:
HashMap<String, String> map = new HashMap<>();
for (Map.Entry<String,String> entry : map.entrySet()){
    String key = entry.getKey();
    String value = entry.getValue();
}

    sb.append(i);
}

07 若需频繁调用Collection.contains方法则使用Set #

反例

// 频繁调用 Collection.contains() 反例
List<Object> list = new ArrayList<>();
for (int i = 0; i <= Integer.MAX_VALUE; i++){
    //时间复杂度为O(n)
    if (list.contains(i))
        System.out.println("list contains "+ i);
}

正例

// 频繁调用Collection.contains() 正例
List<Object> list = new ArrayList<>();
Set<Object> set = new HashSet<>();
for (int i = 0; i <= Integer.MAX_VALUE; i++){
    // 时间复杂度为O(1)
    if (set.contains(i)){
        System.out.println("list contains "+ i);
    }
}

        return new PasswordUtils(aPassword).encrypt();
    }
}

10 删除多余的异常捕获并抛出 #

// 多余异常反例
private static String fileReader(String fileName)throws IOException{

    try (BufferedReader reader = new BufferedReader(new FileReader(fileName))) {
        String line;
        StringBuilder builder = new StringBuilder();
        while ((line = reader.readLine()) != null) {

        }

}

15 枚举的属性字段必须是私有且不可变 #

反例

public enum SwitchStatus {
    // 枚举的属性字段反例
    DISABLED(0, "禁用"),
    ENABLED(1, "启用");

    public int value;
    private String description;

    private SwitchStatus(int value, String description) {
        this.value = value;
        this.description = description;
    }

    public String getDescription() {
        return description;
    }

    public void setDescription(String description) {
        this.description = description;
    }
}

正例

public enum SwitchStatus {
    // 枚举的属性字段正例
    DISABLED(0, "禁用"),
    ENABLED(1, "启用");

    // final 修饰
    private final int value;
    private final String description;

    private SwitchStatus(int value, String description) {
        this.value = value;
        this.description = description;
    }

    // 没有Setter 方法
    public int getValue() {
        return value;
    }

    public String getDescription() {
        return description;
    }
}

16 String.split(String regex)部分关键字需要转译 #

反例

// String.split(String regex) 反例
String[] split = "a.ab.abc".split(".");
System.out.println(Arrays.toString(split));   // 结果为[]

String[] split1 = "a|ab|abc".split("|");
System.out.println(Arrays.toString(split1));  
// 结果为["a", "|", "a", "b", "|", "a", "b", "c"]

正例

// String.split(String regex) 正例
// . 需要转译
String[] split2 = "a.ab.abc".split("\\.");
System.out.println(Arrays.toString(split2));  // 结果为["a", "ab", "abc"]

// | 需要转译
String[] split3 = "a|ab|abc".split("\\|");
System.out.println(Arrays.toString(split3));  // 结果为["a", "ab", "abc"]

17 开发原则 #

  1. 规范命名。见名知意、可读出来的英文命名。
  2. 规范代码格式。合适的空格、代码对齐、及时换行。
  3. 写好代码注释。代码无法看出来，就需要注释了
    • 解释代码的意图，说明为什么这么写，用来做什么
    • 对参数和返回值注释，入参出参代表什么
    • 有警示作用，比如说入参不能为空
    • 当代码还未完成时可使用 todo 标记
  4. try catch 内部代码抽成一个方法。
  5. 方法别太长
  6. 抽取重复代码
  7. 多用 return
  8. if 条件表达式不要太复杂
  9. 参数校验。可单独抽方法或者在实体类上加注解。
  10. 统一返回值
  11. 统一异常处理
  12. 尽量不传递 null 值。
// 示例1
public void updatePerson(@Nullable Person person) {
    if (person == null) {
        return;
    }
    personService.updateById(person);
}

// 示例2
public void updatePerson(@NonNull Person person) {
    personService.updateById(person);
}
  1. 日志打印规范
    • 可搜索性，要有明确的关键字信息。
    • 异常日志需要打印出堆栈信息。
    • 合适的日志级别，比如异常使用error，正常使用info
    • 日志内容太大不打印
  2. 统一类库
  3. 尽量使用工具类，不要重复造轮子
  4. 类和方法单一职责
  5. 使用设计模式优化代码
  6. 经常重构旧代码
  7. null 值判断
  8. pojo 类重写 toString 方法
  9. 魔法值用常量表示
private static final String GUANG_DONG_PROVINCE = "广东省";
public void sayHello(String province) {
    if (GUANG_DONG_PROVINCE.equals(province)) {
        System.out.println("靓仔~~");
    } else {
        System.out.println("帅哥~~");
    }
}
  1. 资源释放写到finally
  2. 慎用异步。事务问题、CPU 负载过高
  3. 减小锁的范围
  4. 有类型区分时定义好枚举
  5. 远程接口调用设置超时时间
  6. 集合使用应当指明初始化大小
  7. 尽量不要使用BeanUtils来拷贝属性
  8. @Transaction应指定回滚的异常类型
  9. 需要什么字段select什么字段
  10. 不循环调用数据库。正确的方法应该批量查出来，然后转成Map
  11. 用业务代码代替多表 join
  12. 装上代码检查插件
  13. 拆分合理的目录结构
  14. 封装方法形参。当一个方法形参过多时请封装一个对象出来。
  15. 注解使用@ConfigurationProperties代替@Value

18 使用线程池代替手动创建线程池 #

  • 降低资源销耗。通过重复利用已创建的线程降低线程创建和销毁造成的消耗。
  • 提高响应速度。当任务到达时，任务可以不需要等到线程创建就能立即执行。
  • 提供线程的可管理性。线程是稀缺资源，如果无限制的创建，消耗系统资源并降低系统的稳定性，使用线程池可进行统一分配，调优和监控。

19 使用聚合/组合代替继承 #

通过成员变量的方式来使用类。

  1. 组合是当对象一创建的时候，就直接给属性赋值。
public class OrderService {
	// 组合
    private UserService userService = new UserService();
}
  1. 聚合可通过set方式来设置。
public class OrderService {
    private UserService userService;
    // 聚合
    public void setUserService(UserService userService) {
        this.userService = userService;
    }
}

20 谨慎方法内部调用动态代理的方法 #

@Service
public class PersonService {
    
    public void update(Person person) {
        // 处理
        updatePerson(person);
    }

    @Transactional(rollbackFor = Exception.class)
    public void updatePerson(Person person) {
        // 处理
    }
}
  1. update调用了加了@Transaction注解的updatePerson方法，那么此时updatePerson的事务就是失效。
  2. 其实失效的原因不是事务的锅。是由AOP机制决定的，因为事务是基于AOP实现的。
  3. AOP是基于对象的代理，当内部方法调用时，走的不是动态代理对象的方法，而是原有对象的方法调用，如此就走不到动态代理的代码，就会失效。 如果实在需要让动态代理生效，可注入自己的代理对象。
@Service
public class PersonService {

    @Autowired
    private PersonService personService;

    public void update(Person person) {
        // 处理
        personService.updatePerson(person);
    }

    @Transactional(rollbackFor = Exception.class)
    public void updatePerson(Person person) {
        // 处理
    }

}

21 映射数据库的属性尽量不要用基本类型 #

基本类型数据作为成员变量默认值是0，在进行插入或者更新的时候很容易会带着默认值插入更新到数据库。

22 封装判断条件 #

public void method(LoanAppEntity loan, long operatorId) {
    if (!loan.finished()) {
        //...
        return;
    }
}

/**
* 贷款单是否完成
*/
public boolean finished() {
    return LoanAppEntity.LoanAppStatus.OVERDUE != this.getStatus()
        && LoanAppEntity.LoanAppStatus.CURRENT != this.getStatus()
        && LoanAppEntity.LoanAppStatus.GRACE_PERIOD != this.getStatus();
}

24 list iterator #

  • 推荐
Iterator<String> iterator = list.iterator();
while (iterator.hasNext()) {
    String item = iterator.next();
    if (删除元素的条件) {
      iterator.remove();
    }
}

iterator

iterator 的特点是更加安全，因为它可以确保，在当前遍历的集合元素被更改的时候， 就会抛出 ConcurrentModificationException 异常。
