Maven #

坐标属性 #

Maven 坐标为各种组件引入了秩序，任何一个组件都必须明确定义自己的坐标。

<!-- https://mvnrepository.com/artifact/org.springframework.boot/spring-boot-starter-parent -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.3.12.RELEASE</version>
    <type>pom</type>
</dependency>
名称备注
groupId定义当前Maven项目隶属的实际项目-公司名称，jar 包所在仓库路径，由于 Maven 中模块的概念，因此一个实例项目往往会被划分为很多模块。
artifactId该元素定义了实际项目中的一个 Maven 模块-项目名，推荐的做法是实际项目名称作为artifactId的前缀。
version该元素定义 Maven 项目当前所处的版本。
type依赖的类型，大部分情况下不需要声明，默认值为 jar。
scope依赖范围，compile：编译依赖范围（默认），test：测试依赖范围（如：junit），provided：已提供依赖范围（如：servlet-api），runtime：运行的依赖范围（如：jdbc 驱动程序），system：系统依赖范围（一般不使用）。
optional标记依赖是否可选
exclusions用来排除传递性依赖

版本

SNAPSHOT：快照，开发版

alpha：内部测试

beta：公开测试

Release | RC：发布版

GA：正常版本

依赖优先原则 #

https://zhuanlan.zhihu.com/p/83236129

  1. 依赖最短路径优先：一个项目Demo依赖了两个jar包，其中A-B-C-X(1.0) ， A-D-X(2.0)。由于X(2.0)路径最短，所以项目使用的是X(2.0)。
  2. pom 文件中申明顺序优先：如果A-B-X(1.0) ，A-C-X(2.0) 这样的路径长度一样怎么办呢？这样的情况下，maven会根据pom文件声明的顺序加载，如果先声明了B，后声明了C，那就最后的依赖就会是X(1.0)。
  3. 覆盖优先原则：子pom内声明的优先于父pom中的依赖。
